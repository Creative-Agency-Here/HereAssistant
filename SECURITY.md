# Security Policy

## Модель угроз: это шлюз удалённого выполнения кода

HereAssistant по замыслу — мост «Telegram → CLI-агент (claude/codex/gemini) на
твоей машине». Агент читает и правит файлы, запускает команды. Любой, кто
получил доступ к боту, получает возможности агента на хосте. Относись к
развёртыванию соответственно.

## Критические точки

- **Доступ к Telegram-боту.** `ADMIN_IDS` — единственный список допущенных.
  Токен бота (`TELEGRAM_BOT_TOKEN`) = возможность перехватить управление.
- **Auth-файлы провайдеров.** `.runtime/cli_homes/<аккаунт>/` содержит
  OAuth-креды подписок Claude/OpenAI/Google. Не копируй, не коммить, бэкапь
  только шифрованно.
- **`WEBAPP_ACCESS_KEY`.** Полный доступ владельца к WebApp из браузера без
  Telegram. Генерируй длинный случайный (`openssl rand -hex 32`), не логируй,
  не вставляй в URL при передаче другим людям.
- **`WEBAPP_DEV_SKIP_AUTH=1` в проде ЗАПРЕЩЁН** — это полное отключение
  авторизации WebApp.
- **`SERVICE_API_TOKEN`.** Даёт внешней системе доступ к `/api/v1/*`, но не
  обходит privacy-политику проектов (private/local невидимы). Пустой токен =
  API отключён.
- **История и журналы.** Если проект разрешил хранение
  (`.hereassistant/project.yml`), в `bridge.sqlite3` и `.runtime/logs/`
  оказываются prompt'ы, ответы и диффы — там могут быть секреты. По умолчанию
  хранение выключено (см. `docs/privacy.md`).
- **Права CLI.** `CLAUDE_PERMISSION_MODE=bypassPermissions` не используем:
  авто-одобрение Bash/Write превращает любой prompt-injection в RCE.

## Рекомендации по развёртыванию

- API слушает только `127.0.0.1`; наружу — nginx с HTTPS.
- Отдельный системный пользователь без sudo; секреты — только в `.env` (0600).
- Обновляй зависимости (`pip`, `npm audit`) и следи за релизами CLI-провайдеров.

## Как сообщить об уязвимости

Не открывай публичный issue. Напиши владельцу репозитория напрямую (контакты в
профиле) с описанием, шагами воспроизведения и версией. Мы отвечаем в разумный
срок и координируем раскрытие после фикса.
